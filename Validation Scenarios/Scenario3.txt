Scenario 3: LLM Accuracy, Dual-Level RAG, & Cache Stress Test

Objective: Stress-test the local LightRAG system's accuracy on multi-hop logical questions, and verify that the LLM pipeline successfully utilizes the Two-Tier Semantic Query Cache and Ollama KV Cache to handle high-frequency requests without memory starvation.

Pre-conditions: A dense, 30-page technical PDF (with cross-referenced concepts) has been ingested into the local LightRAG database, extracting entities and relations.

Test Execution:

    Accuracy Test: Ask a complex relational question requiring dual-level retrieval (e.g., "How does the sensor array described in chapter 2 connect to the logic board in chapter 5?").

    Exact Cache Hit (Tier 1): Immediately ask the exact same question again.

    Context Reuse Cache (Tier 2): Ask a semantically similar question (e.g., "Explain the connection between the logic board and the sensors").

Expected Outcome:

    Accuracy: The LLM uses both local entity-graph retrieval and global vector similarity to provide a highly accurate, hallucination-free answer combining chapters 2 and 5.

    Tier 1 Stress Pass: The second identical query evaluates against the semantic cache (AURA_CACHE_EXACT_THRESH), skipping retrieval/generation entirely and returning the answer instantly via TTS.

    Tier 2 Stress Pass: The third query evaluates above the context threshold (AURA_CACHE_CTX_THRESH). It skips the FAISS database search, re-feeds the cached context to the LLM, and successfully triggers the Ollama KV Cache—dropping prefill token processing from ~3000 tokens down to ~30 tokens for rapid inference. The device passes the stress test by avoiding VRAM out-of-memory errors.
