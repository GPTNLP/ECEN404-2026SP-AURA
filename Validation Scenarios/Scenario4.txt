Scenario 4: Edge-to-Cloud Orchestration & Offline Resilience

Objective: Verify stateful syncing between the Jetson and Azure, ensuring that if the network drops during operation, no telemetric data or chat logs are lost.

Test Execution:

    Disconnect the Jetson's Wi-Fi network.

    Ask the robot a local question to generate a chat log. Trigger a movement command to generate an event log.

    Reconnect the Jetson's Wi-Fi network.

Expected Outcome:

    During the offline period, the API calls will fail, but the send_or_queue_log function will save the chat data and telemetry locally into the pending_logs.jsonl queue.

    Upon network restoration, the background flush_loop() will automatically detect the connection and flush the queued statuses and logs to the Website backend, proving zero-data-loss architecture at the edge.
