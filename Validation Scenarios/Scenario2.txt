Scenario 2: Acoustic & Intent Routing Validation (The Agent Layer)

Objective: Verify that the faster-whisper STT loop accurately captures speech, filters out background noise, and uses the local Ollama intent parser to correctly route commands versus academic questions.

    Pre-conditions: The microphone is connected, Ollama is running (llama3.2:3b), and the environment has a moderate amount of ambient noise (e.g., a typical lab).

    Test Execution:

        Remain silent for 10 seconds to test ambient noise rejection.

        Speak clearly into the microphone: "AURA, please move backward."

        Speak clearly into the microphone: "Can you explain the difference between a mesh network and a star topology?"

    Expected Outcome:

        The terminal should not print any [STT] Transcribed: logs during the silence (the >5 character length filter should reject static).

        The first phrase should be classified as MOVEMENT by the Ollama intent parser and automatically trigger the ESP32 backward sequence.

        The second phrase should be classified as QUESTION and routed to the rag_manager.query() function, outputting a text response to the UI instead of triggering the motors.
